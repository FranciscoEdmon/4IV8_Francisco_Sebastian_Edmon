Formulario y validación

Todos los formularios que están en internet tienen metodos de programación por defecto. Estos metodos no sirven para poder hacer las acciones principales como son: Acciones para poderse conectar con los servidores, a traves de los servidores, atraves de las Arquitecturas Cliente servidor. El metodo que vamos a tener por defecto para todos los formularios utilizan el método GET, y el segundo método principal es POST, aunque existen otros métodos que se encargan del intercambio de información.

  • - GET: este metodo envia por medio de la URL toda la información del formulario a traves del navegador. Su funcion principal es obtener los datos del servicio; pero es el metodo más vulnerable de todos, derivado a que se pueden generar inyecciones de código.
  • - POST: Este metodo se encarga de registar una nueva entrada al sistema, tambien utiliza la URL como envio por defecto; pero su facultad es poder ocultarla, esto significa que los datos no son visibles para el usuario comun en el navegador.
